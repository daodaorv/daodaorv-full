# 叨叨房车管理端功能需求文档

**版本**: v3.0.0 | **日期**: 2025-10-21 | **类型**: 纯业务功能需求

---

## 1. 产品概述

### 1.1 产品定位

叨叨房车管理端是面向运营团队的后台管理系统，为小程序端提供内容配置、规则管理、业务审核、数据分析等支撑服务。

**核心价值**：

- 灵活配置小程序端所有展示内容
- 统一管理业务规则和价格策略
- 完整的审核流程和订单处理
- 实时数据监控和业务分析

### 1.2 核心管理模块

| 管理模块       | 功能描述                                 | 小程序端对应             |
| -------------- | ---------------------------------------- | ------------------------ |
| **用户管理**   | 用户信息管理、资料审核、标签管理         | 用户注册、认证、会员     |
| **车辆管理**   | 车型模板、车辆信息、状态控制、评价管理   | 房车列表、详情、预订     |
| **订单管理**   | 订单处理、退款管理、异常处理、执行监控   | 订单流程、状态跟踪       |
| **营地管理**   | 营地信息、预订管理、评价管理             | 营地列表、预订           |
| **众筹管理**   | 项目创建、进度管理、收益分配、份额交易   | 众筹列表、众筹车辆、分润 |
| **优惠券管理** | 优惠券创建、发放、使用统计               | 优惠券购买、使用         |
| **社区管理**   | 内容审核、用户互动、话题管理             | 社区内容、互动           |
| **客服管理**   | 客服系统配置、工作台、会话管理、订单执行 | 在线客服、帮助中心       |
| **内容配置**   | 首页配置、常见问题、规则条款、重要提示   | 首页、帮助、规则页面     |
| **价格策略**   | 动态定价、因子配置、策略组合             | 房车价格展示             |
| **分润管理**   | 分润规则、佣金管理、提现审核             | 推广分享、收益查看       |
| **财务管理**   | 收支管理、对账结算、财务报表             | 支付、退款               |
| **数据统计**   | 业务数据分析、用户行为分析、运营报表     | -                        |

### 1.3 业务目标

- 建立完整的配置管理体系
- 优化审核和订单处理流程
- 完善数据统计和风控体系
- 实现智能化、自动化运营

## 4. 核心业务管理功能

### 4.1 用户管理功能

#### 4.1.1 用户列表管理

**功能描述**：

- 查看所有注册用户信息
- 多维度筛选和搜索用户
- 批量操作用户状态
- 导出用户数据报表

**筛选条件**：

- 用户状态：全部/正常/冻结/封禁
- 会员类型：普通用户/PLUS 会员/众筹车主
- 注册时间：今日/本周/本月/自定义时间段
- 资料状态：已提交/未提交/审核中/审核通过/审核拒绝

**列表字段**：

- 用户头像、昵称、手机号（脱敏）
- 会员类型、注册时间
- 资料状态（实名、驾照）
- 账户状态、最后登录时间
- 操作按钮：查看详情、编辑、状态管理

**业务规则**：

- 手机号脱敏显示：138\*\*\*\*8888
- 普通管理员只能查看，不能编辑
- 运营管理员可以编辑基本信息
- 超级管理员可以修改所有信息

#### 4.1.2 用户资料审核

**实名资料审核**：

- 审核内容：真实姓名、电话号码、身份证号码、身份证正反面照片
- 审核标准：照片清晰、信息真实、格式正确
- 审核结果：通过或拒绝（需填写原因）

**驾照资料审核**：

- 审核内容：驾驶证号码、驾驶证正副页照片、电话号码
- 审核标准：照片清晰、号码格式正确、在有效期内
- 审核结果：通过或拒绝

**业务规则**：

- 资料审核仅检查格式正确性，不进行在线验证
- 实际验证在线下交付时进行
- 审核拒绝需填写详细原因
- 用户可重新提交资料

#### 4.1.4 用户标签管理

**功能描述**：为用户添加业务标签，用于精准营销和权益管理。

**标签类型**：

- **系统标签**：PLUS 会员、众筹车主、加盟商、分销商、员工
- **行为标签**：高频租车、优质客户、投诉用户
- **自定义标签**：运营人员可自定义标签

**标签应用**：

- 推广分享收益计算
- 精准营销和消息推送
- 用户分群和数据分析
- 特殊权益和服务

**业务规则**：

- PLUS 会员标签：用户付费开通后自动添加
- 众筹车主标签：用户众筹项目众筹车辆正式上线运营时自动添加
- 其他标签：运营人员手动添加或系统自动识别
- 标签可以叠加，一个用户可拥有多个标签

#### 4.1.5 常用人管理

**功能描述**：查看和管理用户添加的常用人信息。

**查看内容**：

- 常用人姓名、电话、身份证号
- 驾照信息（如已添加）
- 关系标签（本人/家人/朋友）
- 添加时间、使用次数

**管理操作**：

- 查看用户的所有常用人
- 查看常用人使用记录
- 协助用户处理常用人问题

**业务规则**：

- 每个用户最多 10 个常用人
- 常用人资料由用户自行管理
- 管理端只提供查看功能

### 4.2 车辆管理功能

#### 4.2.1 功能概述

**业务目标**：管理车型模板和具体车辆信息，确保车辆信息准确、状态实时、评价真实。

**管理范围**：

- 车型模板管理（标准化车型信息）
- 具体车辆管理（实际车辆档案）
- 车辆状态实时管理
- 车辆评价审核和回复

#### 4.2.2 车型模板管理

**功能描述**：创建和管理标准化的车型模板，为具体车辆提供基础信息。

**车型模板内容**：

- **基本信息**：车型名称、品牌、型号、分类（B 型/C 型/拖挂式）
- **车辆参数**：座位数、床位数、车身尺寸、整备质量
- **设施配置**：厨房、卫浴、空调、冰箱、电视等设施清单
- **车型图片**：外观图、内饰图、设施图、全景图
- **车型详情**：富文本编辑的详细介绍、使用说明、特色亮点
- **默认价格**：基础日租价、周租价、月租价、押金

**业务规则**：

- 车型模板作为标准配置，具体车辆引用模板
- 门店添加车辆时可在模板基础上勾选实际配置
- 车型详情支持富文本编辑，显示在小程序详情页
- 车型图片和详情由内容管理员维护

**小程序端对应**：

- 车型详情页的"车辆图文详情"区域显示车型详情
- 车辆配置列表显示车型配置信息

#### 4.2.3 具体房车管理

**功能描述**：管理实际房车的档案信息、状态、维护记录。

**房车档案信息**：

- **基本信息**：车牌号、VIN 码、关联车型模板、所属门店
- **所有权信息**：平台自有/众筹房车/
- **房车状态**：可用/已租/维护中/停用
- **实际配置**：基于车型模板勾选的实际配置
- **价格来源**：使用车型默认价格或价格策略管理

**房车维护记录**：

- 维护时间、维护内容、维护费用
- 里程数、油量、车况评估
- 维护人员、维护门店

**房车调度记录**：

- 调度时间、调出门店、调入门店
- 调度原因、费用分摊
- 原始归属门店（保持不变）

**业务规则**：

- 房车必须关联车型模板
- 房车价格统一使用车型默认价格或价格策略
- 房车调度后原始归属不变，用于收益分配
- 维护中的房车不显示在小程序端

#### 4.2.4 房车评价管理

**功能描述**：审核用户提交的房车评价，回复用户评价，管理评价展示。

**评价审核**：

- 查看待审核评价列表
- 检查评价内容是否合规（无违禁词、无恶意攻击）
- 检查评价图片是否真实合规
- 审核通过或拒绝

**评价回复**：

- 平台可对用户评价进行官方回复
- 回复内容显示在评价下方
- 用于解释说明或致谢致歉

**评价聚合规则**：

- 评价按车型模板聚合显示
- 同一车型的所有房车评价汇总展示
- 评分计算：所有评价的平均分
- 评价排序：最新评价优先

**业务规则**：

- 评价必须审核通过才显示
- 评价一经发布不可修改
- 恶意评价可删除并说明原因
- 评价聚合让用户更全面了解车型

**小程序端对应**：

- 房车详情页的"用户评价"区域显示该车型的所有评价

#### 4.2.5 房车标签管理

**功能描述**：为房车配置展示标签，突出房车特色和优势。

**标签类型**：

- **系统标签**：

  - 新车：车龄小于 1 年自动标记
  - 热门：近 30 天预订量排名前 20%自动标记
  - 推荐：人工设置，突出推荐房车
  - 特价：参与特惠活动的房车自动标记

- **自定义标签**：
  - 运营人员可自定义标签
  - 例如：网红车型、高端车型、家庭首选等
  - 每辆房车最多显示 3 个标签

**标签配置**：

- 标签名称
- 标签颜色（用于小程序端显示）
- 标签图标
- 显示优先级
- 是否启用

**业务规则**：

- 系统标签自动生成，不可删除
- 推荐标签需人工设置
- 标签按优先级显示，最多显示 3 个
- 标签配置后实时同步到小程序

**小程序端对应**：

- 房车列表卡片显示标签
- 房车详情页显示标签
- 标签突出房车优势，促进转化

### 4.3 订单管理功能

#### 4.3.1 功能概述

**业务目标**：管理所有订单流程，处理订单异常，审核特殊申请，监控订单执行。

**管理范围**：

- 订单列表查询和筛选
- 订单详情查看
- 订单状态管理
- 订单异常处理（退款、续租、事故）
- 订单执行监控（取还车）

#### 4.3.2 订单列表管理

**功能描述**：查看和管理所有类型订单。

**订单类型**：

- 房车租赁订单
- 特惠租车订单
- 营地预订订单
- 定制旅游订单

**筛选条件**：

- 订单状态：全部/待支付/待取车/使用中/待还车/已完成/已取消/已退款
- 订单类型：房车/特惠/营地/旅游
- 下单时间：今日/本周/本月/自定义
- 取车门店：所有门店/指定门店
- 用户信息：手机号/姓名搜索

**列表字段**：

- 订单号、订单类型、订单状态
- 用户信息（昵称、手机号）
- 车辆信息/营地信息
- 租期/入住期、订单金额
- 取车门店、创建时间
- 操作按钮：查看详情、处理

**业务规则**：

- 门店经理只能查看本门店订单
- 门店员工只能查看本门店订单
- 客服人员可查看全部订单
- 财务人员可查看全部订单

#### 4.3.3 订单退款处理

**功能描述**：处理用户的退款申请，审核退款金额，执行退款操作。

**退款申请审核**：

1. 查看退款申请列表
2. 查看退款详情（订单信息、退款原因、退款金额）
3. 核实订单状态和取消时间
4. 根据取消政策计算应退金额
5. 审核通过或拒绝（拒绝需说明原因）
6. 审核通过后系统自动退款

**退款金额计算**：

- 取车前 24 小时以上：全额退款
- 取车前 24 小时内：扣除 10%手续费
- 取车后退款：根据实际使用天数计算

**业务规则**：

- 退款申请必须财务管理员审核
- 退款金额根据取消政策自动计算，可手动调整
- 退款原路返回原支付方式
- 退款记录完整保留备查

#### 4.3.4 续租申请审核

**功能描述**：审核用户的续租申请，确认车辆可用性，调整订单和费用。

**续租审核流程**：

1. 查看续租申请列表
2. 查看申请详情（原订单、续租天数、续租原因）
3. 检查车辆续租期间是否有预订冲突
4. 计算续租费用
5. 审核通过或拒绝
6. 通知用户审核结果和费用
7. 用户支付续租费用
8. 更新订单还车时间

**续租条件**：

- 必须在原订单还车日期前提交申请
- 续租期间车辆无其他预订冲突
- 用户无订单违约或事故欠费记录
- 续租价格按当前价格策略计算

**业务规则**：

- 续租申请必须提前 24 小时
- 续租最长不超过原订单租期
- 续租价格可能与原订单价格不同（根据当前策略）
- 续租费用单独支付，不合并原订单

#### 4.3.5 事故报告处理

**功能描述**：处理用户报告的车辆事故，协调保险理赔，记录事故处理过程。

**事故处理流程**：

1. 接收用户事故报告
2. 查看事故详情（时间、地点、现场照片、事故描述）
3. 联系用户了解详情
4. 判断事故严重程度（轻微/中度/严重）
5. 指导用户处理（报警/保险定损/现场等待）
6. 协调保险公司理赔
7. 评估责任和费用
8. 记录事故处理过程
9. 更新订单状态和费用

**事故分类**：

- **轻微事故**：轻微剐蹭，不影响使用，维修费<1000 元
- **中度事故**：部分损坏，需维修，维修费 1000-5000 元
- **严重事故**：严重损坏或人员伤亡，需报警和保险处理

**责任判定**：

- 用户责任：用户操作不当导致的事故
- 平台责任：车辆本身质量问题
- 不可抗力：自然灾害、第三方责任等

**业务规则**：

- 事故必须在 24 小时内报告
- 严重事故必须报警和保险理赔
- 责任判定影响用户赔偿金额
- 保险范围内的损失由保险赔付

### 4.4 众筹管理功能

#### 4.4.1 功能概述

**业务目标**：管理众筹项目全生命周期，从项目创建、众筹管理、收益分配到项目结束。

**管理范围**：

- 众筹意向处理
- 房车验车和评估
- 众筹项目创建和发布
- 众筹记录管理

| 项目         | 说明                                                                  |

#### 4.4.6 收益分润管理

**功能描述**：管理众筹项目的收益分配，按月计算和发放分润。

**分润计算**：

```
月租赁收入 = 该房车所有订单收入总和
月运营成本 = 保险费 + 维护费 + 清洁费 + 平台服务费
月净收益 = 月租赁收入 - 月运营成本
个人分润 = 月净收益 × 份额比例
```

**分润流程**：

1. 每月 1 日系统自动计算上月收益
2. 财务管理员审核收益计算
3. 确认无误后批量发放分润
4. 分润直接转入众筹车主钱包
5. 发送分润通知给所有众筹车主

**分润规则**：

- 按月分润，每月 10 日前发放
- 分润自动转入用户钱包
- 提供详细的收益明细
- 亏损月份不分润

**小程序端对应**：

- 用户在"我的份额"查看持有车辆和收益
- 查看每月分润明细和历史记录

### 4.5 份额市场管理功能

#### 4.5.1 功能概述

**业务目标**：管理众筹份额的二级交易，审核挂牌申请，监控交易过程，处理交易纠纷。

**管理范围**：

- 挂牌出售审核
- 交易过程监控
- 份额过户处理
- 交易手续费配置
- 交易纠纷处理

#### 4.5.2 挂牌出售审核

**功能描述**：审核众筹车主提交的份额挂牌出售申请。

**审核内容**：

- 核实卖家持有份额情况
- 检查份额是否可转让（无冻结、无纠纷）
- 核对挂牌价格合理性
- 确认同一车辆份额整体转让

**审核标准**：

- 份额真实存在且属于申请人
- 价格在合理范围内（参考价 ±50%）
- 同车份额整体转让，不拆分
- 无其他限制条件

**审核流程**：

1. 查看挂牌申请列表
2. 查看申请详情
3. 核实份额信息
4. 审核通过或拒绝
5. 审核通过后份额上架市场

**业务规则**：

- 挂牌需要审核，防止异常交易
- 审核时限：24 小时内完成
- 拒绝需说明原因
- 份额上架后随时可撤销

#### 4.5.3 交易监控管理

**功能描述**：实时监控份额交易过程，及时发现和处理异常。

**监控内容**：

- 交易订单列表
- 交易状态（待支付/已支付/过户中/已完成/已取消）
- 交易双方信息
- 交易金额和手续费
- 资金流转记录

**异常预警**：

- 异常高价或低价交易
- 频繁交易同一份额
- 交易双方关联账号
- 支付异常或超时

**业务规则**：

- 所有交易需要监控
- 异常交易及时人工介入
- 保护买卖双方权益

#### 4.5.4 份额过户处理

**功能描述**：处理份额所有权转移，更新份额持有记录。

**过户流程**：

1. 买家支付成功后自动触发过户
2. 系统扣除卖家份额
3. 系统增加买家份额
4. 更新份额持有记录
5. 计算并扣除手续费
6. 转账给卖家（扣除手续费后）
7. 发送过户通知给买卖双方

**过户记录**：

- 过户时间

- 手续费金额
- 过户前后份额持有情况

**业务规则**：

- 过户完成后不可撤销
- 完整记录备查

**配置内容**：

- 最高手续费：例如 1000 元

**业务规则**：

- 手续费从卖家收款中扣除
- 手续费配置实时生效

#### 4.5.6 交易纠纷处理

**功能描述**：处理份额交易中的纠纷和投诉。

**纠纷类型**：

- 支付纠纷：已支付未到账
- 过户纠纷：过户延迟或失败
- 价格纠纷：价格显示错误
- 欺诈纠纷：虚假交易

**处理流程**：

1. 接收纠纷投诉
2. 调查核实情况
3. 联系双方了解详情
4. 查看交易记录和证据
5. 做出处理决定
6. 执行处理方案（退款/补偿/警告）
7. 记录处理过程
8. 通知双方处理结果

**业务规则**：

- 纠纷必须在 7 天内提出
- 平台保持中立，依据事实处理
- 恶意投诉将受到处罚
- 所有处理有完整记录

### 4.9 增值费用管理功能

#### 4.9.1 费用项管理

**配置项**：费用名称、类型（固定/可选/结算/保证金）、计算方式（固定金额/按天/按比例/按距离/按阶梯）、费用金额、费用归属（平台/门店/客户）、是否参与分润、启用状态

**关键功能**："是否参与分润"勾选框决定费用是否计入分润基数

#### 4.9.2 费用分类

**费用类型**：

- 固定费用：门店服务费、保险费、车辆管理费
- 可选费用：儿童座椅、GPS 导航、送车上门、异地还车
- 结算费用：超时费、清洁费、燃油/电费差额、违章罚款、车辆维修费
- 保证金：车辆押金（不参与分润）

#### 4.9.3 费用计算方式

- 固定金额：每单固定金额
- 按天计算：单价 × 天数
- 按比例计算：订单金额 × 比例
- 按距离计算：距离 × 公里单价
- 按阶梯计算：根据条件收取不同金额
- 按时长计算：按小时阶梯收费

#### 4.9.4 异地还车费管理

- 归属平台
- 门店可申请实际产生的调度费用
- 申请时限：车辆归还后 30 天内
- 发放上限：不超过收取的异地费用

#### 4.9.5 分润基数计算

1. 订单总额 = 租金 + Σ(参与分润的增值费用)
2. 优惠后金额 = 订单总额 - 优惠券抵扣
3. 可分润金额 = 优惠后金额 - Σ(不参与分润的费用)
4. 各方分润 = 可分润金额 × 配置的分润比例

#### 4.9.6 业务规则

- 费用归属在订单完成后触发
- 费用调整后对新订单立即生效
- 勾选"参与分润"的费用计入分润基数
- 保证金固定不参与分润
- 订单退款时所有费用原路退回
- 结算费用需门店还车时确认
- 异常费用需管理员审核
- 异地还车费用申请需审核后发放

#### 5.2.6 价格预览测试

- 选择车辆、城市、日期，系统自动计算最终价格

#### 5.4.1 功能概述

#### 5.4.2 综合排序权重配置

**功能描述**：配置房车列表综合排序的权重系数。

**权重因子**：

| 因子名称       | 权重范围 | 默认权重 | 说明                 |
| -------------- | -------- | -------- | -------------------- |
| **房车评分**   | 0-100    | 30       | 用户评价的综合评分   |
| **预订热度**   | 0-100    | 25       | 近期预订次数和浏览量 |
| **价格合理性** | 0-100    | 20       | 价格与同类车型的对比 |
| **车辆新旧**   | 0-100    | 15       | 车龄和里程数         |
| **可用性**     | 0-100    | 10       | 可用日期数量         |

**计算公式**：

```
综合得分 = 房车评分×权重1 + 预订热度×权重2 + 价格合理性×权重3 + 车辆新旧×权重4 + 可用性×权重5
```

**配置方式**：

- 调整各因子权重（总和=100）
- 实时预览排序效果
- 一键恢复默认权重
- 保存后立即生效

**业务规则**：

- 所有权重之和必须等于 100
- 权重调整后影响所有房车列表
- 提供 AB 测试功能，对比不同权重效果
- 记录权重调整历史

**小程序端对应**：

- 房车列表默认按综合排序展示
- 排序结果根据配置的权重计算

**推荐规则**：

**配置项**：

**业务规则**：

#### 5.6.6 特惠数据统计

**功能描述**：统计特惠套餐的销售数据和活动效果，为运营决策提供依据。

| **平均客单价** | 销售额 ÷ 订单数   |
| **库存销售率** | 已售数量 ÷ 总库存 |

**套餐排行**：

- 销量最高的套餐 Top10
- 收入最高的套餐 Top10
- 库存销售率 Top10

**时间趋势**：

- 按日统计订单量和销售额

**城市分布**：

- 按取车城市统计订单量
- 按还车城市统计订单量

**用户分析**：

- 新用户占比
- 老用户复购率
- 用户来源渠道

**活动效果评估**：

| 评估指标         | 计算方式            |
| ---------------- | ------------------- |
| **转化率**       | 订单数 ÷ 访问量     |
| **销售完成率**   | 已售数量 ÷ 总库存   |
| **平均优惠幅度** | (原价-特惠价)÷ 原价 |
| **ROI**          | 销售额 ÷ 运营成本   |

### 5.10 分润管理功能

#### 5.10.1 分润基数配置

**可分润产品配置**：

| 产品类型   | 是否参与分润 | 分润基数   | 说明                 |
| ---------- | ------------ | ---------- | -------------------- |
| 房车日租金 | ☑️ 是        | 租金金额   | 核心收入来源         |
| 营地预订   | ☑️ 是        | 订单金额   | 营地服务费           |
| 旅行产品   | ☑️ 是        | 订单金额   | 定制旅游费用         |
| 增值服务费 | 根据配置     | 服务费金额 | 在增值费用管理中配置 |

**不可分润费用配置**：

在"增值费用管理"功能（4.9 章节）中，每个费用项都有"是否参与分润"勾选框：

- 勾选"☑️ 否"的费用：不计入分润基数，直接划归归属方
- 勾选"☑️ 是"的费用：计入分润基数，参与分润计算

**分润基数计算公式**：

```
步骤1：计算订单总额
订单总额 = 租金 + Σ(勾选"参与分润☑️是"的增值费用)

步骤2：扣除优惠
优惠后金额 = 订单总额 - 优惠券抵扣

步骤3：扣除不参与分润的费用
不参与分润费用 = Σ(勾选"参与分润☑️否"的费用)
可分润金额 = 优惠后金额 - 不参与分润费用

步骤4：各方分润
根据用户标签配置的分润比例计算各方收益
```

**优惠券处理规则**：

- 优惠券抵扣不影响分润基数计算
- 优惠成本由平台承担
- 分润基于优惠后的实际支付金额

#### 5.10.3 标签分润配置管理

**配置总览页面**：

显示所有参与分润的标签及其配置：

| 标签名称     | 标签类别 | 分润类型 | 分润比例         | 分润层级 | 操作   |
| ------------ | -------- | -------- | ---------------- | -------- | ------ |
| 加盟商       | 门店运营 | 门店分润 | 10%              | 一级     | [配置] |
| 企业分销商   | 门店运营 | 推广分润 | 10%              | 一级     | [配置] |
| PLUS 会员    | 推广权益 | 推广分润 | 一级 5%，二级 1% | 二级     | [配置] |
| 众筹车主     | 众筹身份 | 车主分润 | 30%              | 一级     | [配置] |
| 特殊推广用户 | 自定义   | 推广分润 | 自定义           | 一级     | [配置] |

**分润配置说明**：

- **加盟商/企业分销商**：一级分润，比例可配置（默认 10%）
- **PLUS 会员**：二级分销
  - 会员推广奖励：一级 20 元/单，二级 10 元/单（可配置）
  - 订单推广奖励：一级 5%，二级 1%（可配置）
- **众筹车主**：车主分润 30%（可配置），按份额比例分配
  - 支持虚拟车主"平台"

#### 5.10.4 分润叠加规则配置

- 不同类别的标签分润可叠加
- 同一订单同标签只能获得一次分润
- 分润总额无上限（超高比例需预警）

#### 5.10.5 分润计算与记录

**触发时机**：订单状态变更为"已完成"

**计算流程**：

1. 计算可分润金额（扣除不参与分润的费用）
2. 识别分润方（门店、车辆众筹车主、推广人）
3. 按标签配置比例计算各方分润
4. 生成分润明细记录
5. 发放到用户账户

- 优惠券抵扣：根据管理端设置的叠加规则

- 年龄要求：21-70 岁

- 资料要求：需提交驾照资料

**续租规则**：

- 申请时间：还车前 24 小时
- 续租价格：按当前价格策略计算
- 车辆可用性：无后续预订冲突

- 众筹资格：需提交实名资料

**收益分配规则**：

- 分润周期：按月
