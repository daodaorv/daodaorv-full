## 1. 产品概述

### 1.1 产品定位

叨叨房车小程序是一个专注于房车租赁服务的移动端应用，为用户提供便捷的房车预订、营地预订、定制旅游、社区互动等一站式房车旅行服务。

**核心价值主张**：

- **便捷预订**：随时随地查看房车信息，快速完成预订
- **透明价格**：明码标价，费用清晰透明
- **安全保障**：多重认证，保险完备，用车无忧
- **社区互动**：房车爱好者交流分享，增强用户粘性
- **创新模式**：众筹模式，让众筹车主共享房车租赁收益

**城市服务逻辑**：

- **本地化服务**：房车租赁、营地预订、定制旅游基于用户所在城市提供服务
- **自动定位**：首次进入自动获取用户所在城市，展示该城市相关产品
- **城市切换**：用户可手动切换城市，查看其他城市的服务

### 1.2 核心业务模块

| 业务模块       | 业务价值                                 | 目标用户               |
| -------------- | ---------------------------------------- | ---------------------- |
| **房车租赁**   | 提供多样化房车租赁服务，满足不同场景需求 | 个人旅行者、家庭出游   |
| **特惠租车**   | 固定路线特价服务，降低出行成本           | 价格敏感用户、新手用户 |
| **营地预订**   | 解决房车旅行住宿问题                     | 房车租赁用户           |
| **定制旅游**   | 个性化路线定制，专业服务                 | 高端用户、特殊需求用户 |
| **优惠券购买** | 提供租车优惠，促进复购                   | 活跃用户、会员用户     |
| **社区互动**   | 增强用户粘性，促进 UGC 内容产生          | 房车爱好者、分享达人   |
| **众筹房车**   | 提供房车众筹和收益分润服务               | 众筹车主               |
| **用户中心**   | 提供完整的用户服务体验                   | 所有用户               |

### 1.3 业务目标

**短期目标（3-6 个月）**：

- 用户注册量达到 10,000 人
- 月活跃用户达到 3,000 人
- 房车租赁订单量达到 500 单/月
- 用户满意度达到 4.5 分以上（5 分制）

**中期目标（6-12 个月）**：

- 拓展服务城市至 20 个
- 平台房车数量达到 500 辆
- 众筹项目成功率达到 80%以上
- 复购率达到 30%以上

**长期目标（1-3 年）**：

- 成为国内领先的房车租赁平台
- 构建完整的房车旅行生态圈
- 实现盈亏平衡并持续盈利

## 2. 用户角色与场景

### 2.1 用户角色定义

| 用户角色      | 角色特征                       | 核心需求                     | 使用场景             |
| ------------- | ------------------------------ | ---------------------------- | -------------------- |
| **普通用户**  | 对房车旅行感兴趣，偶尔租车     | 方便租车、价格优惠、使用简单 | 节假日出游、周末郊游 |
| **PLUS 会员** | 高频租车用户，愿意付费享受特权 | 专属折扣、优先服务、免费取消 | 频繁租车、长期租赁   |
| **众筹车主**  | 看好房车租赁市场，愿意参与众筹 | 稳定回报、资金安全、流动性   | 资产配置、被动收入   |

### 2.2 典型用户场景

#### 场景 1：家庭周末郊游

**用户**：张先生，35 岁，有两个孩子
**需求**：周末带家人去郊区露营
**使用流程**：

1. 周四晚上打开小程序，自动定位深圳，查看可用房车
2. 浏览房车列表（约 20-30 辆可选），查看详情和评价
3. 选择合适的家庭型房车，预订周六上午 10 点到周一上午 10 点（租期 2 天）
4. 在线支付，获得订单确认
5. 周六上午 10 点到门店取车，开始旅行
6. 周一上午 10 点还车，完成订单评价

**说明**：租期按 24 小时为 1 天，取还车时间点相同，最低租期 2 天

**期望收益**：

- 便捷预订，10 分钟内完成
- 价格透明，费用可预期
- 车辆状况良好，符合家庭出游需求

#### 场景 2：参与众筹项目

**用户**：王先生，众筹爱好者
**需求**：寻找稳定的众筹渠道
**使用流程**：

1. 浏览众筹项目列表
2. 查看项目详情和收益预测
3. 选择购买 10 份，共 30,000 元
4. 提交实名资料，签署众筹协议，完成支付
5. 等待众筹结果
6. 众筹成功，成为众筹车主
7. 每月收到收益分润
8. 根据需要在份额市场转让份额

**期望收益**：

- 年化收益 12-18%，高于银行理财
- 众筹风险可控，有实物资产支撑
- 流动性好，可在份额市场交易

## 4. 核心业务流程

### 4.1 房车租赁完整业务流程

#### 4.1.1 流程概述

房车租赁是平台的核心业务，完整流程包括：浏览选择 → 预订确认 → 服务履行 → 订单完成 四个阶段。

#### 4.1.2 详细流程步骤

**第一阶段：浏览选择**

**业务目标**：帮助用户快速找到合适的房车

| 步骤 | 用户操作                   | 系统功能                       | 业务价值           |
| ---- | -------------------------- | ------------------------------ | ------------------ |
| 1.1  | 进入首页，查看房车预订模块 | 展示快速预订入口               | 降低用户操作门槛   |
| 1.2  | 选择城市、门店、时间       | 提供城市门店选择器和日期选择器 | 明确租赁条件       |
| 1.3  | 点击查询，进入房车列表     | 根据条件查询可用房车           | 展示符合条件的结果 |
| 1.4  | 浏览房车列表（综合排序）   | 按综合排序展示可用房车         | 推荐最合适的房车   |
| 1.5  | 点击房车卡片，查看详情     | 展示房车完整信息、图片、评价   | 帮助用户决策       |
| 1.6  | 查看可用性和价格           | 实时显示房车可预订情况         | 确保信息准确       |

**用户痛点**：

- 不知道选什么车型合适
- 不清楚价格是否合理
- 担心车辆状况不好

**解决方案**：

- 综合排序智能推荐合适的房车
- 提供详细的车辆参数和配置说明
- 展示真实用户评价和评分
- 提供多角度高清图片
- 显示明确的价格明细

**第二阶段：预订确认**

**业务目标**：确保用户顺利完成预订，信息准确完整

| 步骤 | 用户操作                   | 系统功能                         | 业务价值     |
| ---- | -------------------------- | -------------------------------- | ------------ |
| 2.1  | 点击"立即预订"按钮         | 进入预订流程                     | 开始预订     |
| 2.2  | 确认取车信息（地点、时间） | 显示用户选择的信息，可修改       | 确保信息准确 |
| 2.3  | 填写联系人信息             | 收集姓名、手机号                 | 便于联系沟通 |
| 2.4  | 选择保险险种               | 展示多种保险选项和保障范围       | 降低用车风险 |
| 2.5  | 选择附加服务               | 展示可选服务（如导航、儿童座椅） | 满足额外需求 |
| 2.6  | 填写特殊需求               | 提供备注输入框                   | 个性化服务   |
| 2.7  | 签署电子合同               | 展示租赁合同，用户签署           | 明确双方权责 |
| 2.8  | 确认订单信息和价格         | 显示完整订单摘要和费用明细       | 确保用户知情 |
| 2.9  | 选择支付方式，完成支付     | 提供多种支付方式                 | 完成交易     |

**用户痛点**：

- 不了解保险区别，不知道选哪个
- 担心有隐藏费用
- 合同条款太长看不懂

**解决方案**：

- 用通俗语言解释各保险的区别
- 明确展示所有费用明细，无隐藏收费
- 高亮显示合同重要条款
- 提供合同条款解释说明

---

**第三阶段：服务履行**

**业务目标**：确保用户顺利取车、用车、还车

| 步骤 | 用户操作                     | 系统功能                          | 业务价值     |
| ---- | ---------------------------- | --------------------------------- | ------------ |
| 3.1  | 支付成功后收到订单确认       | 发送订单确认通知（站内消息+短信） | 让用户安心   |
| 3.2  | 查看订单详情，了解取车流程   | 提供订单详情页，显示取车指引      | 明确取车流程 |
| 3.3  | 按约定时间到门店取车         | 订单中提供门店地址和导航          | 便于找到门店 |
| 3.4  | 现场验车、签署纸质合同、取车 | 记录验车情况，上传系统            | 明确车辆状况 |
| 3.5  | 使用期间遇到问题             | 提供 24 小时客服和紧急救援        | 解决用车问题 |
| 3.6  | 需要续租                     | 提供续租申请功能                  | 灵活延长租期 |
| 3.7  | 发生事故                     | 提供事故报告功能                  | 及时处理事故 |

**用户痛点**：

- 不知道取车流程，第一次使用很迷茫
- 用车中遇到问题不知道找谁
- 需要延长租期不知道怎么办

**解决方案**：

- 提供详细的取车指引和注意事项
- 订单页面显示客服联系方式
- 提供续租申请功能，流程简单
- 提供事故报告入口，引导用户处理

---

**第四阶段：订单完成**

**业务目标**：确保顺利还车，收集用户反馈

| 步骤 | 用户操作               | 系统功能               | 业务价值     |
| ---- | ---------------------- | ---------------------- | ------------ |
| 4.1  | 按约定时间到门店还车   | 提供还车提醒和注意事项 | 确保按时还车 |
| 4.2  | 现场验车，确认车辆状况 | 记录还车时车辆状况     | 明确责任     |
| 4.3  | 等待押金退还           | 自动处理押金退还       | 快速退还押金 |
| 4.4  | 订单完成，收到通知     | 发送订单完成通知       | 告知订单结束 |
| 4.5  | 对服务进行评价         | 提供评价功能，收集反馈 | 改进服务质量 |

**用户痛点**：

- 不知道还车时要注意什么
- 担心押金退还慢或不退
- 对服务不满意没有反馈渠道

**解决方案**：

- 还车前发送提醒，列明注意事项
- 明确押金退还时间和条件
- 提供便捷的评价功能
- 对差评及时跟进处理

#### 4.1.3 特殊流程：续租申请

**触发场景**：用户在租期内需要延长租赁时间

**业务流程**：

1. 用户在订单详情页点击"申请续租"
2. 选择续租天数，系统计算续租费用
3. 填写续租原因
4. 提交申请
5. 后台审核（检查车辆后续预订情况）
6. 审核通过，用户支付续租费用
7. 订单租期自动延长

**业务规则**：

- 必须在当前租期结束前 24 小时申请
- 续租费用按当前价格标准计算
- 如车辆后续已被预订，则无法续租
- 续租申请需人工审核确认

#### 4.1.4 特殊流程：事故处理

**触发场景**：租赁期间发生交通事故或车辆损坏

**业务流程**：

1. 用户在订单页点击"事故报告"
2. 填写事故信息（时间、地点、经过）
3. 上传现场照片和相关证件
4. 提交报告
5. 系统立即通知客服和门店
6. 客服联系用户，指导现场处理
7. 根据情况安排救援或维修
8. 后台评估责任和费用
9. 处理保险理赔
10. 用户支付应承担的费用
11. 根据情况决定订单是否继续

**业务规则**：

- 事故发生后必须立即报告
- 严重事故需报警处理
- 保险理赔按保险条款执行
- 用户承担保险免赔额部分
- 用户违反租赁协议导致订单无法继续的，平台有权终止订单

### 4.2 众筹项目业务流程

#### 4.2.1 流程概述

众筹项目是平台的创新业务，平台采购或整合房车资源，通过众筹方式让用户购买份额，共同分享租赁收益。

#### 4.2.2 详细流程步骤

**第一阶段：项目筹备**

**业务目标**：平台准备优质房车资源，设计合理的众筹方案

| 步骤 | 操作方 | 操作内容                      | 业务价值         |
| ---- | ------ | ----------------------------- | ---------------- |
| 1.1  | 平台   | 采购或整合优质房车资源        | 提供众筹标的     |
| 1.2  | 平台   | 评估车辆价值和运营潜力        | 确保众筹可行性   |
| 1.3  | 平台   | 设置总份额数（通常 100 份）   | 便于众筹车主参与 |
| 1.4  | 平台   | 设置单份价格（评估价值 ÷100） | 降低参与门槛     |
| 1.5  | 平台   | 预测月收益和年化收益率        | 吸引众筹车主     |
| 1.6  | 平台   | 设置众筹期限（通常 30 天）    | 控制募集时间     |
| 1.7  | 平台   | 设置最低成功份额（通常 80%）  | 确保项目可行     |

**众筹方案示例**：

- 车辆评估价值：300,000 元
- 总份额：100 份
- 单份价格：3,000 元
- 预计月收益：5,000-9,000 元
- 预计年化收益率：12-18%
- 众筹期限：30 天
- 最低成功份额：80 份（240,000 元）

---

**第二阶段：公开众筹**

**业务目标**：吸引众筹车主参与，完成众筹目标

| 步骤 | 操作方   | 操作内容                     | 业务价值     |
| ---- | -------- | ---------------------------- | ------------ |
| 5.1  | 平台     | 众筹项目上线，对所有用户开放 | 开始募集资金 |
| 5.2  | 众筹车主 | 浏览众筹项目列表             | 寻找众筹机会 |
| 5.3  | 众筹车主 | 查看项目详情和收益预测       | 评估众筹价值 |
| 5.4  | 众筹车主 | 选择购买份额数量             | 决定购买金额 |
| 5.5  | 众筹车主 | 签署众筹协议                 | 明确权责     |
| 5.6  | 众筹车主 | 支付购买款                   | 完成购买     |
| 5.7  | 系统     | 实时更新众筹进度             | 展示众筹状态 |

**众筹结果**：

- **成功**：达到最低份额或售罄，项目进入运营
- **失败**：众筹期结束未达最低份额，全额退款

---

**第三阶段：运营分润**

**业务目标**：车辆运营产生收益，按份额分润

| 步骤 | 操作方   | 操作内容                     | 业务价值     |
| ---- | -------- | ---------------------------- | ------------ |
| 6.1  | 平台     | 车辆上线运营，接受租赁订单   | 产生收益     |
| 6.2  | 平台     | 每月统计租赁收入和成本       | 计算净收益   |
| 6.3  | 平台     | 按份额比例计算各众筹车主分润 | 公平分配     |
| 6.4  | 平台     | 每月 10 日前发放上月分润     | 及时支付     |
| 6.5  | 众筹车主 | 查看收益明细和分润记录       | 了解收益情况 |
| 6.6  | 众筹车主 | 随时可在份额市场转让份额     | 提供流动性   |

**收益计算公式**：

```
月租赁收入 = 所有订单收入总和
月运营成本 = 保险费 + 维护费 + 清洁费 + 平台服务费
月净收益 = 月租赁收入 - 月运营成本
个人分润 = 月净收益 × 份额比例
```

**运营成本项目说明**：

| 成本项目       | 说明                         | 计算方式             |
| -------------- | ---------------------------- | -------------------- |
| **保险费**     | 车辆保险费用                 | 按月固定或按订单比例 |
| **维护费**     | 车辆定期维护和保养费用       | 按月平均分摊         |
| **清洁费**     | 每次租赁后的专业清洁费用     | 按订单数计算         |
| **平台服务费** | 平台提供的运营支持和服务费用 | 按租赁收入的比例     |

**分润规则**：

- 按月分润，每月 10 日前发放
- 分润直接打入用户钱包
- 提供详细的收益明细
- 亏损月份不分润
- 分润计算公式与管理端保持一致

### 4.3 份额市场交易流程

#### 4.3.1 流程概述

份额市场为众筹项目的众筹车主提供份额转让服务，提高份额流动性。

#### 4.3.2 挂牌出售流程

**业务目标**：让众筹车主能够快速转让份额

| 步骤 | 卖家操作             | 系统功能                 | 业务价值     |
| ---- | -------------------- | ------------------------ | ------------ |
| 1    | 进入"我的份额"页面   | 展示持有的所有份额       | 查看资产     |
| 2    | 选择要出售的车辆份额 | 展示份额详情和收益情况   | 了解份额价值 |
| 3    | 点击"挂牌出售"       | 进入出售设置页面         | 开始出售流程 |
| 4    | 设置出售价格         | 提供定价建议和市场参考价 | 合理定价     |
| 5    | 确认出售信息         | 展示手续费和到账金额     | 明确费用     |
| 6    | 提交挂牌申请         | 后台审核                 | 防止异常交易 |
| 7    | 审核通过后份额上架   | 份额在市场展示           | 开始售卖     |

**定价建议**：

- 原价出售：快速成交
- 溢价 5-15%：合理溢价，考虑累计收益
- 溢价 15%+：需等待合适买家

**手续费规则**：

| 手续费项目       | 说明                                      |
| ---------------- | ----------------------------------------- |
| **基础手续费率** | 成交金额的 3%                             |
| **最低手续费**   | 10 元（成交金额较小时按最低手续费收取）   |
| **最高手续费**   | 1000 元（成交金额较大时按最高手续费收取） |
| **VIP 用户优惠** | PLUS 会员享受手续费 8 折优惠              |
| **收费方式**     | 从卖家收款中扣除                          |

**手续费计算示例**：

- 成交金额 ¥500：手续费 = max(¥500×3%, ¥10) = ¥15
- 成交金额 ¥100：手续费 = max(¥100×3%, ¥10) = ¥10（按最低手续费）
- 成交金额 ¥50,000：手续费 = min(¥50,000×3%, ¥1000) = ¥1000（按最高手续费）
- VIP 用户成交 ¥50,000：手续费 = ¥1000 × 80% = ¥800

##### 5.1.3.4 业务规则

**城市门店规则**：

1. 城市列表仅显示有服务门店的城市
2. 城市按热门程度排序，支持搜索
3. 切换城市后，门店列表自动更新
4. 取还车门店不同时收取异地还车费，费用在订单确认页展示

**时间选择规则**：

1. 取车时间最早为当前时间+4 小时（后台可配置）
2. 取车时间最晚为当前时间+6 个月
3. 还车时间必须晚于取车时间
4. **最短租赁时长为 2 天（48 小时）**
5. 最长租赁时长为 60 天（后台可配置）
6. **租赁天数按 24 小时为 1 天计算，取还车时间点必须相同**
7. 时间选择器仅显示营业时间段（后台配置）

**默认值规则**：

1. 优先使用用户上次选择的城市和门店
2. 如无历史选择，使用用户定位城市
3. 如无法定位，使用默认城市（深圳）
4. 取车时间默认为当前时间+4 小时，取整点（如当前 14:30，默认 18:00）
5. 还车时间默认为取车时间+2 天，时间点同步（如取车 10:00，还车也是 10:00）

**异地还车规则**：

1. 支持异地还车的城市由后台配置
2. 异地还车需支付额外费用（具体金额在确认页显示）
3. 部分车型不支持异地还车

### 5.6 营地预订功能需求

- 完善的营地信息展示，帮助用户选择合适的营地

- **双模式运营**：营地支持实时预订或咨询模式，由管理端在营地列表页快速切换
- **第三方对接**：对接优质营地资源，扩大服务覆盖

- 选择营位类型（房车营位-带水电）
- 填写入住信息（车牌号、人数）

### 5.7 特惠租车功能需求

#### 5.7.1 功能概述

**业务定位**：限时特惠的房车租赁服务，提供固定路线、固定租期的套餐化租车方案，以优惠价格吸引价格敏感用户。

**业务目标**：

- 提供比普通租赁更优惠的价格，降低用户出行成本
- 通过限时促销提升订单转化率
- 推广特定路线和目的地，引导用户出行
- 简化预订流程，提供标准化套餐服务

**核心特点**：

- **固定路线**：预设路线组合，如"伊宁 → 乌鲁木齐"
- **固定租期**：限定租期天数，如 5 天、7 天
- **套餐价格**：两种定价方式（套餐总价或特惠日租）
- **限时优惠**：活动期内有效，超期恢复原价
- **免费异地还车**：支持 A 地取 B 地还，无异地还车费
- **套餐化服务**：仅含基础租赁，服务保障和附加服务需额外选择

**与普通房车租赁的区别**：

| 维度           | 普通房车租赁                 | 特惠租车                              |
| -------------- | ---------------------------- | ------------------------------------- |
| **选择自由度** | 自由选择任意车辆、时间、租期 | 限定车辆、时间段、固定租期            |
| **价格模式**   | 按日租价格 × 天数计算        | 套餐总价或特惠日租价                  |
| **时间选择**   | 任意时间，最短 2 天          | 限定时间段内，固定租期（如必须 5 天） |
| **异地还车**   | 需支付异地还车费             | 免费异地还车（限定路线）              |
| **库存来源**   | 所有可用车辆                 | 限定的活动车辆（批量采购）            |
| **产品形态**   | 租赁服务                     | 限时促销套餐                          |

### 5.8 定制旅游功能需求

#### 5.8.1 功能概述

**业务定位**：提供标准化房车旅游套餐，包含固定路线、固定行程、专业服务的多日游产品。

**业务目标**：

- 提供一站式房车旅游服务，用户无需操心行程规划
- 通过标准化套餐降低运营成本，提升服务质量
- 满足用户省心省力的旅游需求
- 提供自驾和管家随行两种模式，满足不同用户需求

**核心特点**：

- **标准化套餐**：预设固定路线和行程，用户只需选择出发日期
- **完整服务**：包含房车、住宿、餐饮、门票等一站式服务
- **双模式选择**：自驾模式或管家随行模式
- **按人头计费**：价格透明，按成人/儿童收费
- **在线预订**：类似跟团游，选择日期和人数即可预订

**与其他业务的区别**：

| 维度         | 普通房车租赁   | 特惠租车          | 定制旅游                 |
| ------------ | -------------- | ----------------- | ------------------------ |
| **产品性质** | 租赁服务       | 促销套餐          | 旅游产品                 |
| **服务范围** | 仅车辆         | 车辆+免费异地还车 | 车辆+住宿+餐饮+门票+导游 |
| **行程安排** | 用户自行规划   | 固定路线，无行程  | 固定路线+详细行程        |
| **服务期限** | 2-60 天        | 固定租期(5-10 天) | 固定行程(3-15 天)        |
| **价格模式** | 日租价 × 天数  | 套餐价            | 按人头收费               |
| **用户群体** | 会开房车的用户 | 价格敏感用户      | 想省心的旅游用户         |

#### 6.1.2.1 押金规则详细说明

**押金组成与支付方式：**

| 押金类型 | 金额 | 支付方式 | 支付时间 | 退还条件 | 退还时间 |
|---------|------|---------|---------|---------|---------|
| **车辆押金** | ¥5,000（可配置） | 线下扫码支付 | 取车时门店支付 | 车辆无损归还 | 订单完成后立即退还 |
| **违章押金** | ¥5,000（可配置） | 线下扫码支付 | 取车时门店支付 | 30日内无违章记录 | 订单完成后30日退还 |

**支付流程说明：**

1. **线上预授权**：下单时无需支付押金，系统自动记录押金金额
2. **线下支付**：用户到店取车时，由门店工作人员发起支付邀请
3. **扫码支付**：用户通过微信/支付宝扫码完成押金支付
4. **收据确认**：支付成功后，系统生成电子收据

**退还规则：**

- **车辆押金退还**：还车验车完成后，系统自动发起退还流程
- **违章押金退还**：30天违章查询期结束后，如无违章记录自动退还
- **扣除说明**：如发生车辆损坏或违章，相应押金将被扣除，差额部分退还

**免押金服务（预留功能）：**

系统已预留免押金接口，后期将支持：
- 支付宝芝麻信用免押金
- 微信支付分免押金
- 会员等级免押金优惠
- 特殊活动免押金政策

#### 7.0.3 登录流程

**微信一键登录流程**:

```
用户点击"微信一键登录"
  ↓
检查用户协议勾选
  ├─ 未勾选 → 提示"请先同意用户协议和隐私政策"
  └─ 已勾选 → 继续
       ↓
调用 uni.login() 获取 code
  ↓
调用 uni.getUserInfo() 获取用户信息(可选)
  ↓
调用后端 POST /api/auth/wechat-login
  ├─ 参数: { code, nickname, avatar }
  └─ 返回: { token, user }
       ↓
保存 token 到 localStorage 和 Pinia store
  ↓
跳转到原目标页面或首页
```

**支付宝一键登录流程**:

调用 my.getAuthCode() 获取 authCode
  ↓
调用后端 POST /api/auth/alipay-login
  ├─ 参数: { authCode }
  └─ 返回: { token, user }

**抖音一键登录流程**:

#### 7.0.5 Token 管理

**Token 存储**:

- localStorage: 持久化存储,刷新页面后仍然有效
- Pinia store: 内存存储,用于快速访问

**Token 使用**:

- 所有需要认证的接口请求头携带: `Authorization: Bearer <token>`
- Token 有效期: 7 天
- Token 过期后自动跳转到登录页

**Token 刷新** (可选):

- 在 Token 即将过期时自动刷新
- 或在 Token 过期后引导用户重新登录

#### 7.0.6 登录拦截

**路由拦截器**:

- 在 `miniprogram/utils/router.ts` 中实现
- 跳转前检查目标页面的权限要求
- 未登录用户访问需要登录的页面时,自动跳转到登录页
- 登录成功后自动返回原目标页面

**权限配置**:

- 在 `miniprogram/config/page-auth.ts` 中配置每个页面的权限要求
- 权限等级: `PUBLIC` (公开) | `LOGIN_REQUIRED` (需要登录)

#### 7.0.7 业务规则

1. **用户协议**: 登录前必须勾选用户协议和隐私政策
2. **平台检测**: 根据运行平台动态显示对应的一键登录按钮
3. **验证码限制**:
   - 同一手机号 60 秒内只能发送一次验证码
   - 验证码有效期 5 分钟
   - 验证码错误次数限制: 5 次
4. **登录回跳**: 登录成功后自动跳转到原目标页面或首页
5. **多平台账号**:
   - 同一手机号可以绑定多个平台账号
   - 不同平台登录的用户可以通过手机号关联为同一账号

### 7.1 用户权限体系

#### 7.1.1 权限等级定义

| 权限等级         | 用户类型   | 主要权限             | 限制说明                   |
| ---------------- | ---------- | -------------------- | -------------------------- |
| **游客权限**     | 未登录用户 | 浏览信息             | 无法预订、购买、互动       |
| **基础权限**     | 已登录用户 | 预订、购买、社区互动 | 微信/支付宝/手机号登录即可 |

- 资料提交在用户执行对应操作时引导完成，不影响浏览和其他功能使用
- 所有资料在线下交付时进行实物核验
